Leetcode Contest 197

Posted on 2020-07-11 In 算法

本次周赛关键字: 滑动窗口, BFS, Dijkstra, 迭代法

Easy: Number of Good Pairs

暴力解就不说了. 计算每一个数字出现的数字, 假如一个数字出现了n次那么它就可以组成n * (n-1) / 2个好数对

举例假如1出现了3次 ... 1 ... 1 ... 1.... 那么我们可以任意取两个1组成好数对,实际上答案就是 1 + 2 + 3 + ... + (n-1)

时间复杂度: O(n)
空间复杂度: O(1)

Medium: Number of Substrings With Only 1s

  • 滑动窗口做法

用连续的1来构建滑动窗口

  1. 如果当前值为0, windowStart和windowEnd同时变为下一次循环的下标
  2. 在连续的1的窗口中时,当前窗口可以构建的子字符串为end - start + 1

时间复杂度: O(n)
空间复杂度: O(1)

  • lee215

第一种做法可以抽象为计算连续的1的数量. 用count来记录连续的1, 对于每个新元素都会有count个新的子字符串.

时间复杂度: O(n)
空间复杂度: O(1)

Medium: Path with Maximum Probability

  • BFS
  1. 用double[] cache来记录start到每个节点的最大概率. 用Map<Integer, Map<Integer, Double>> map记录所有的边以及边的权重(概率)
  2. 初始化cache和map并建立一个队列来开始BFS. 队列中初始只有start
  3. BFS每次遍历时我们遍历所有当前节点的邻节点并计算出从当前节点到达邻节点的概率newNextProb
    1. 假如newNextProb小于等于cache[next], 即一个我们之前计算过的到达邻节点的最大概率, 直接抛弃这个邻节点(不加入队列)
    2. 假如newNextProb大于cache[next], 即我们找到了一条比已经探索过的路径更好的路径. 更新cache[next]并把next加入队列
  4. 返回cache[end]

时间复杂度: O(E2)
空间复杂度: O(V)

  • Dijkstra’s algorithm

前一种BFS做法很容易让复杂度接近超时, 更正确的做法应该是把最大概率问题转化为最短路径问题. 使用基于优先队列的Dijkstra算法可以把复杂度降低到O(E * log(V)).

时间复杂度: O(E * log(V))
空间复杂度: O(V)
