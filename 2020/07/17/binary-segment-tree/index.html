线段树及其应用

Posted on 2020-07-17 Edited on 2020-07-18 In 算法

线段树

什么是线段树

线段树(Segment Tree)是一种平衡二叉树形数据结构(保证树的高度为log(n)), 用来存储区间或线段的性质, 并且可以快速查询结构内包含某一点的所有区间. 在线段树中, 每个节点都代表一个区间. 通常每个节点会存储一个或者多个关于这个区间性质的信息. 在本文中我们只讨论离散线段树在数组中的应用.

为什么需要线段树?

很多问题需要我们根据一段数据的查询求解, 这是一个很慢的过程, 尤其是当数据量很大的时候. 线段树可以让我们在对数级的时间复杂度内完成这一操作. 在现实生活中线段树常用在计算几何和GIS中. 比如当我们有很多点在一个平面中, 我们需要找到距离原点一定距离之内的所有点. 这种操作通常会需要我们遍历所有的点, 而线段树可以很高效地解决这种问题. 这类问题称作Planar Range Searching.

线段树的结构


假设我们有一个数组[2, 1, 5, 3, 4]并将它存入线段树中. 对于每一个节点我们都存储了该节点对应的区间[i,j]以及对应该区间的某种或多种性质. 在图中我们选择了存储对应区间的数值和, 我们当然也可以存储别的性质类似于最大值, 最小值等等.

线段树的实现及复杂度

节点及构造树

用start和end来记录区间起点和终点的闭区间.
用sum记录当前区间的数值和.
最后用left和right记录左子树和右子树.

  1. 假如start == end, 证明当前区间只有一个元素, 因此我们已经构建到了线段树的叶子节点. 直接返回一个没有左右子树的SegmentTreeNode
  2. 每次递归都将区间减半. 先找到中点mid, 把start到mid传递到左子树的构造函数, 把mid + 1到end传递到右子树的构造函数.
  3. 返回新的SegmentTreeNode, sum为左节点sum加右节点sum.

时间复杂度: BuildTree T(n) = 2 * T(n / 2) = O(n)

更新树

线段树的一大特点是可以动态更新树中的元素

  1. 如果当前节点的start和end都与需要更新的节点下标index相等的话, 证明当前节点即为需要更新的叶子节点. 直接更新sum返回.
  2. 找到当前区间的终点, 假如index <= mid, 证明要更新的节点属于左子树, 反之属于右子树. 继续进行递归查找.
  3. 最后不要忘记更新路径上的sum

时间复杂度: UpdateTree: T(n) = T(n/2) + O(1) = log(n)

区间查找

  1. 如果当前节点的区间与要查询的区间完全吻合, 直接返回当前节点的sum
  2. 求区间中点, 如果查询区间的终点落在区间中点的左边, 递归查询左子树
  3. 如果查询区间的起点落在区间中点的右边, 递归查询右子树
  4. 如果查询区间横跨区间终点的两边, 在左子树中查找[i, mid], 在右子树中查找[mid + 1, j], 并返回它们的和

时间复杂度: querySum: O(logn + k) k是检查的区间数

例题: Range Sum Query - Mutable

Lazy Propagation 懒传播

我们目前只是在更新单个元素, updateTree的log(n)复杂度很高效. 但是如果我们需要更新一段数据呢? 在当前的方法中, 每个元素都需要被单独更新.
树结构引入了另一个问题叫做ancestral locality. 相邻叶子节点的祖先在树的某一层一定会是同一个, 更新每一个叶子节点意味着我们需要对它们的共同祖先更新多次. 我们能否减少这里的重复计算呢?


在上面这个例子中, 根节点被更新了三次, 而82这个节点被更新了两次. 在树的不同高度, 不同叶子节点的更新会最终更新同一个节点. 在实际应用中的另一个问题是, 当查询的区间不包含经常被更新的元素时, 我们可能会浪费宝贵时间在更新很少被使用的节点上.

Lazy Propagation(懒传播)可以通过减少浪费的运算时间并且根据需求来处理节点来解决这些问题

如何操作?

如名字所说, 我们懒更新节点. 延后更新节点子节点的时间, 直到节点本身被查询到.
现在我们假设更新操作会更新一个范围内的节点, 并且这种方式需要使用数组来表示线段树本身.

为了方便理解本文使用了真正的树节点来做展示, 实际上数组会更快
使用数组表示线段树时基本思路完全一致但我们用一个四倍于原数组大小的数组来表示线段树
为什么是四倍? stack overflow上的一篇回答
根节点在tree[0]的位置
tree[i]的子树在tree[2*i + 1]和tree[2*i + 2]

使用一个和tree[]相同大小的数组lazy[]来代表所有懒节点. lazy[i]包含了当节点i被访问时tree[i]应该增加的值. 假如lazy[i]是0, 这意味着tree[i]已经是最新值且不需要更新.

懒范围更新
  1. 标准化当前节点并移除懒节点的标记. 我们对当前节点进行增加并标记它的子树需要更新.
  2. 如果当前区间在更新区间中就对当前节点进行更新操作
  3. 递归对子节点进行更新操作
查询一颗懒更新的树
  1. 标准化当前节点并移除懒标记. 与懒范围更新中的操作一致.
  2. 递归查询子节点

注意: 下面这些行是针对Range Sum Query - Mutable这道题. 不同的问题可能需要进行不同的更新与合并操作. 在这里更新就是增加+val且节点包含的是区间元素的和
