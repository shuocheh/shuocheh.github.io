动态规划问题(1)-最长上升子序列及其变种

Posted on 2020-07-06 In 算法

题型归纳 credit to: @wh0ami

Longest Increasing Subsequence

给一个无序的整型数组，求最长上升子序列的长度

Input: [10,9,2,5,3,7,101,18]
Output: 4
Explanation: The longest increasing subsequence is [2,3,7,101], therefore the length is 4.

  • 暴力解
  1. 递归函数使用三个参数,数组本身, 上一个元素, 当前坐标
  2. 递归的结束条件为当前坐标到达了数组的末尾
  3. 在每次递归中我们考虑两种情况
    1. 当前元素大于包含在LIS里的上一个元素, 所以我们也把当前元素加入当前LIS
    2. 当前元素小于包含在LIS里的上一个元素,所以我们不能把当前元素加入当前LIS
  4. 比较3中的两种情况,返回较大的

时间复杂度: O(2n) 递归树的size
空间复杂度: O(n2)

  • 优化暴力解

暴力解中的最大问题是我们对同样的参数调用了多次递归函数, 假如我们把递归函数的结果存储在下来,就能让效率大大提高

时间复杂度: O(n2) 递归树的size
空间复杂度: O(n2)

  • 动态规划解

从前两种方法中我们可以看出真正影响结果的其实并不是具体的每个LIS的内容, 我们关心的只有1)当前LIS的结尾元素和2)当前LIS的长度.
可以归纳出递归函数:
dp[i]: 以第i个元素结尾的最长LIS的长度
dp[i]=max(dp[j])+1,∀0≤j<i

时间复杂度: O(n2) 两层循环
空间复杂度: O(n) dp数组长度

  • 贪心+二分搜索

这里有点跑题, 其实我觉得这并不算动态规划,更像一个贪心算法.
基本思路是如果我们要让最长上升子序列尽可能的长, 我们需要让序列的上升速度尽可能慢. 如果希望序列的上升速度尽可能慢,很自然的就能想到每次在上升子序列最后加上的那个数字应该尽可能小.

  1. 建立一个数组dp[i]: 长度为i的最长上升子序列的末位元素的最小值, 并且用len来记录当前的最长上升子序列长度
  2. 循环遍历数组中的所有元素
  3. 利用Java自带的Arrays.binarySearch来要么找到1)当前元素在dp中的下标要么找到2)如果当前元素应该被插入数组它应该被插入的位置
  4. 假如3中的函数返回了负值我们通过i = -(i + 1)求得3中的2)
  5. 更新数组, 如果i的坐标超过了当前最大坐标则len自增

以输入序列 [0,8,4,12,2] 为例：

得到结果为3. 尽管数组中并不存在0 2 12这种子序列,但是这并不会影响结果, 因为我们只关心可能的最大结果

时间复杂度: O(nlog(n))
空间复杂度: O(n)

Largest Divisible Subset

给一个数组,求最大的可以任意两个元素取余都为0的子集

Input: [1,2,4,8]
Output: [1,2,4,8]

这道题的本质和前面的LIS问题一模一样. 不难看出只要数组是升序的, 对于一个子集, 只要新的元素对子集中最大的元素取余为0, 那么新元素就必定对子集中所有元素取余都为0.

dp[i]: 以第i个元素作为最大元素的子集的大小
dp[i]=max(dp[j])+1,∀0≤j<i
由于题目要求返回子集本身,我们维护一个单独的变量index来记录最大子集的结尾数字的坐标, 同时维护一个数组pre, 用pre[index]来记录当前数字坐标在子集里前一个数字的坐标

时间复杂度: O(n2) 两层循环
空间复杂度: O(n) dp数组长度

Russian Doll Envelopes

非常经典的一道题: 俄罗斯套娃信封
用(w,h)来表示一个信封的场合款. 一个信封能且只能在长宽都严格小于另一个信封的时候被套入另一个信封
求最多可以有多少的信封组成俄罗斯套娃

Input: [[5,4],[6,4],[6,7],[2,3]]
Output: 3
Explanation: The maximum number of envelopes you can Russian doll is 3 ([2,3] => [5,4] => [6,7]).

这道题也是很典型的LIS问题, 很容易可以想到用升序排序之后直接进行LIS. 但是这样带来的问题是同样宽度的信封可能在结果中有多个. 可以首先按照宽度递增, 高度递减的方式排序, 之后对高度进行LIS.

这样就可以避免同一宽度下不符合条件的高度被计算多次

这里是LIS的解法4, 为了避免面试不允许直接用自带的Arrays.binarySearch手写了一下

时间复杂度: O(nlog(n))
空间复杂度: O(n)

Maximum Length of Pair Chain

给n个数对
定义假如 b<c 则(c, d)可以连上(a, b)
求最长的数对链长度

Input: [[1,2], [2,3], [3,4]]
Output: 2
Explanation: The longest chain is [1,2] -> [3,4]

纯换皮LIS, 唯一区别就是对原数组进行排序. 这里采用LIS的方法3

时间复杂度: O(n2) 两层循环
空间复杂度: O(n) dp数组长度

Number of Longest Increasing Subsequence

给一个无序的整型数组，求最长上升子序列的数量

Input: [1,3,5,4,7]
Output: 2
Explanation: The two longest increasing subsequence are [1, 3, 4, 7] and [1, 3, 5, 7].

这次不仅要求LIS的长度还要求LIS的数量. 维护第二个数组count来记录当前坐标的LIS的数量.

时间复杂度: O(n2) 两层循环
空间复杂度: O(n) dp数组长度

Longest String Chain

给一个字数串数组
如果我们可以在 word1 的任何地方添加一个字母使其变成 word2，那么我们认为 word1 是 word2 的前身
词链是单词 [word_1, word_2, …, word_k] 组成的序列
求最长词列的长度

Input: [“a”,”b”,”ba”,”bca”,”bda”,”bdca”]
Output: 4
Explanation: “a”,”ba”,”bda”,”bdca”

虽然不如之前明显但是依然是LIS的换皮. 区别在于之前是比较数字大小这次是比较字符串长度

时间复杂度: O(n2)
空间复杂度: O(n)

如何思考DP问题

  1. 思考出一个可能为解的递归关系
  2. 当已经有了一个可能的递归关系时,把递归关系拆分为general case和base case
    1. 在讨论general case时,给dp的递归关系赋予一个具体的含义,比如针对最长上升子序列问题就是dp[i]: 对于以下标i结尾的最长上升子序列长度
    2. 在定义了递推关系函数之后,用dp函数表示原题的返回值. 如果没法表示,就说明递推关系中很可能缺少了某些必要信息
    3. 在top-down(memorization)的题解中base case往往都很简单直接, 因为这就是递归结束的条件
  3. DP问题的本质其实是图问题. 每个dp[...][...][...]其实都可以通过递归关系表示为节点和边的关系. 一个问题如果有DP解法, 那么这个图一定是一个有向无环图. 这常常也是书上说的optimal substructure. 正如费曼所说, 每当在公式中看到π的时候, 就会问: “圆在哪里?” 思考清楚DP问题中的图问题可以帮助我们更深层得理解每种DP问题类型的本质.
