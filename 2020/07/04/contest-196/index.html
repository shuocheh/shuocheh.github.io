Easy: Can Make Arithmetic Progression From Sequence

给一个数组，判断数组是否可以组成一个等差数列

Input: arr = [3,5,1]
Output: true

Input: arr = [1,2,4]
Output: false

先对数组排序，然后判断相邻元素的差是否处处相等

时间复杂度: O(nlogn)
空间复杂度: O(1)

Medium: Last Moment Before All Ants Fall Out of a Plank

有一块木板，长度为 n 个 单位.一些蚂蚁在木板上移动，每只蚂蚁都以 每秒一个单位 的速度移动.其中，一部分蚂蚁向 左 移动，其他蚂蚁向 右 移动.

当两只向 不同 方向移动的蚂蚁在某个点相遇时，它们会同时改变移动方向并继续移动.假设更改方向不会花费任何额外时间.

而当蚂蚁在某一时刻 t 到达木板的一端时，它立即从木板上掉下来.

给你一个整数 n 和两个整数数组 left 以及 right .两个数组分别标识向左或者向右移动的蚂蚁在 t = 0 时的位置.请你返回最后一只蚂蚁从木板上掉下来的时刻.

Input: n = 4, left = [4,3], right = [0,1]
Output: 4

题面很啰嗦, 但是这道题的窍门其实在于当两只蚂蚁相遇的时候, 即使题目中他们会转换方向,在结果上看等价于继续向前

知道了这一点就非常简单了, 我们要求的结果其实就是从初始位置出发, 假设永远不会改变方向, 最后掉下木板的蚂蚁掉下去的时刻

python一行就好

时间复杂度: O(n)
空间复杂度: O(1)

Medium: Count Submatrices With All Ones

给一个只有0和1的矩阵, 求有多少子矩阵的元素全都是1

Input: mat =
[[1,0,1],
[1,1,0],
[1,1,0]]
Output: 13
Explanation:
There are 6 rectangles of side 1x1.
There are 2 rectangles of side 1x2.
There are 3 rectangles of side 2x1.
There is 1 rectangle of side 2x2.
There is 1 rectangle of side 3x1.
Total number of rectangles = 6 + 2 + 3 + 1 + 1 = 13.

对矩阵里的每个点(i, j), 记录终点为j时有多少个连续的1 对每一个点都进行这样的更新存在一个数组dp中. 对于每一个点(i, j)固定它是所有子矩阵的右下角, 利用数组dp从该行向上找[0:i]的行数以`(i,j)为右下角可能形成的矩阵个数. 每次循环都将这个结果加到最终结果中.

举例: 对于
[1 0 1]
[0 1 0]
[1 1 0]
处理后变为
[1 0 1]
[0 1 0]
[1 2 0]
假如我们当前要寻找以(2, 1)为右下角的所有子矩阵
[1 0 1]
[0 1 0]
[1 2 0]
res += dp[2][1]
[1 0 1]
[0 1 0]
[1 2 0]
res += dp[1][1]
可以得到以(2, 1)为右下角的子矩阵有2+1个

时间复杂度O(m2 * n)
空间复杂度O(m * n)

Hard: Minimum Possible Integer After at Most K Adjacent Swaps On Digits

给一个字符串num和一个整数k, 可以交换字符串num中相邻位置的数字最多k次. 返回能得到的最小整数.

Input: num = “4321”, k = 4
Output: “1342”

这道题如果不要求最优解的话唯一考点是贪心, 但是其实这么做是一个O(n2)的暴力解. 假如使用Fenwick Tree可以让复杂度进一步降低到O(nlog(n)). 我只写出了贪心的暴力解, 在给定时间内想到要用并且手写Fenwick Tree对我来说还是太难了.

先来讨论暴力解
想要通过交换数字来让一个很大的整数变得更小, 直觉告诉我们应该尽量把小的数字放到整数的最前面. 我们尽量在k次交换以内把我们可以交换到最前面的最小的数字放到最前,假如这个时候还剩余有交换次数,我们就从下一个位置开始递归得继续进行交换. 这种贪心策略可以通过反证法证明.

  • base case: 假如剩余交换次数k为0或者当前坐标位置已经超出了数组范围,直接返回
  • 遍历[idx:idx + k]的范围,这之内的所有数字都可以在k次交换以内被转移到数组开头. 记录下最小值和它的坐标
  • 不断交换i和i-1位置的数字直到把最小的数字转移到了数组最前面
  • 递归进行下一次循环

时间复杂度O(n2), k非常大的时候实际上就是在进行冒泡排序
空间复杂度O(n)

Fenwick Tree解法: @fanhua

这种解法相比于暴力解的优化在于不需要每次重头开始遍历寻找下一个最小值. 对于每个下标用0或1存在Fenwick Tree里就可以高效判断在一个范围内有多少元素已经被置换过了, 这样可以解决如何计算置换次数的问题.

时间复杂度O(nlog(n))
空间复杂度O(n)
