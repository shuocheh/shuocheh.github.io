Medium: Bulb Switcher IV

这是一道很典型的一看到就应该想到贪心的题. 贪心的思路很简单, 当我们从左到右遍历时, 只要碰到和需要的结果开关不同的位置, 我们就进行一次翻转. 这样一定可以在最少的翻转次数中达到需要的结果.

如何证明贪心?
对于任意的二进制数组, 假设Flip(i)代表从第 i 个位置开始翻转所有数位, Flip(i) -> Flip(j) 与 Flip(j) -> Flip(i) 的结果相同.

Medium: Number of Good Leaf Nodes Pairs

和众多求树中节点距离的题类似, 我们可以用后序遍历来做.
思路: 对于两个叶子节点, 他们的最短距离是他们各自到他们的最小公共祖先的距离的和. 因此很自然可以想到进行后续遍历, 然后在祖先节点的位置处理左右子树中符合条件的点. 这时问题就是我们如何知道能有多少种组合呢? 答案是每次递归时返回一个数组, 用数组来记录不同深度的叶子节点的数量, 然后进行组合运算就好了.

时间复杂度: O(N)
空间复杂度: O(H)

Hard: String Compression II

DFS + 记忆化

用一个四维数组来保存状态 dp[n+1][26][n+1][n+1]
四个维度分别是

  • 当前处理的下标
  • 当前处理的字符
  • 当前处理的字符的长度
  • 剩余的可删除的次数

在每种状态时, 我们有两种选择

  • 保留当前字符
  • 删除当前字符 时间复杂度: O(N3)
    空间复杂度: O(N3)
