看到树的节点是aggregate子树的模式, 首先想到递归

  1. 把边的表现形式转化为图. 由于有些很坑的测试用例(例如[0, 2], [0, 3], [1, 2]), 边的两个节点都要存
  2. 进行dfs递归
    1. 递归函数我们返回一个数组, 数组存放26个字母在子树中的出现次数
    2. 由于我们每条边都存储了两次. 还需要传入一个parent, 这样可以避免在遍历子节点的时候走回头路
    3. base case是如果我们到达了叶子节点, 我们把叶子节点的结果更新为1(没有子树, 只有自己). 并更新count数组并返回
    4. 对于普遍情况, 遍历所有的子节点, 如果遇到了parent就直接进入下一次循环. 根据返回的count数组更新当前count数组的计数
    5. 对当前count数组中当前节点的label自增1
    6. 更新当前节点子树中相同label的数量并返回count数组

时间复杂度: O(26 * N)
空间复杂度: O(N)

Medium???: Maximum Number of Non-Overlapping Substrings

在刚刚结束contest时这道题的AC率不到9%, 这道题主要难度在于思路很难想. 起码我没有想到可以用贪心做, 傻乎乎得在用backtracking尝试遍历所有结果.

这道题有几个要点:

  • 用两个数组firstSeen和lastSeen来记录每个字母第一次出现和最后一次出现的位置
  • 对于两个满足条件的字符串, 如果其中一个是另一个的子串, 优先选择子串(短的那个)
  • 对所有可能的字符串从短到长排序, 贪心地从最短的开始把不重叠的字符串加入结果就是答案(可用反证法证明)

时间复杂度: O(N)
空间复杂度: O(26)

Hard: Find a Value of a Mysterious Function Closest to Target

这道题给定的函数func(arr, l, r)实际上是在求arr[l]到arr[r]的按位与和, 这里分享一下@zerotrac大神的解法

按位与之和有两个特别的性质:

  • 在我们固定一端扩张由l和r构建的窗口时, 结果一定是递减的. 这是因为进行按位与和的操作时, 0永远不可能变成1而1有可能变成0.
  • 还有一条特别的性质是我们的结果范围一定是小于一个固定大小的集合. 因为arr[r] <= 10^6 < 2^24, 所以二进制表示中最多有24个1. 每次进行按位与运算时, 如果这个值发生了变化, 那么会有若干个1变成0. 因此对于固定的r, func(arr, l, r)的值最多只有24种

时间复杂度: O(N)
空间复杂度: O(1)
